    Case agreement rules:

    The following decision tree visualizes the rules used for classifying feature value for a using dependency relation denoted by relation, head-pos and child-pos refer to the POS tag of the head and child token respectively.

    Each node of the tree represents a portion of the data. samples denotes the number of training data points in that node. value is the class distribution within that node. Each edge denotes the feature used for splitting.
    Leaf nodes contain the description of all of the features that appear in that leaf. * denotes that the feature can take any value.

    Given that all feature values for Case are often not equally probable, we use a threshold t to decide the probability of agreement vs non-chance agreement. Click on t=0.9 to toggle between showing/hiding the tree with threshold=0.9

    We evalute the tree on test data along three metrics. Unweighted Distributional Similarity (UDS) measures how the class distribution over the training data matches with the test data for each leaf. Weighted Distributional Similarity (WDS) weighs the UDS score with percent of test data in that leaf. Prediction-Only Distributional Similarity (PDS) measures the UDS score for only the nodes that make a non-random prediction.

      Morphosyntax Rules for Gen:

    1. NOUN tokens need to have dependents with Case = Gen for the dependency relations: nominal modifier (nmod)

    2. NUM tokens need to have Case = Gen, when head token belongs to [NOUN] for the dependency relations: numeric modifier (nummod)

    3. PRON tokens need to have Case = Gen, when head token belongs to [ADP] for the dependency relations: fixed multiword expression (fixed)

    4. NOUN tokens need to have Case = Gen for the dependency relations: numeric modifer governing case of noun (nummod:gov)

      Morphosyntax Rules for Dat:

    1. All tokens have Case=Dat for the dependency relations: indirect object (iobj)

    2. NOUN tokens need to have Case = Dat, when head token belongs to [VERB] for the dependency relations: parataxis (parataxis)

      Morphosyntax Rules for Acc:

    1. All tokens have Case=Acc for the dependency relations: object (obj)

    2. NUM tokens need to have Case = Acc for the dependency relations: numeric modifer governing case of noun (nummod:gov)

    3. ADJ tokens need to have Case = Acc, when head token belongs to [ADP] for the dependency relations: conjunct (conj), fixed multiword expression (fixed)

      Morphosyntax Rules for Loc:

    1. NOUN tokens need to have Case = Loc, when head token belongs to [ADP] for the dependency relations: appositional modifier (appos), conjunct (conj), fixed multiword expression (fixed)

      Morphosyntax Rules for Nom:

    1. NOUN tokens need to have Case = Nom for the dependency relations: passive nominal subject (nsubj:pass), nominal subject (nsubj)

    2. NOUN tokens need to have Case = Nom, when head token belongs to [DET] for the dependency relations: oblique nominal (obl)

      Morphosyntax Rules for Ins:

    1. ADJ tokens need to have Case = Ins, when head token belongs to [VERB] for the dependency relations: oblique nominal (obl)
