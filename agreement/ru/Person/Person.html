    Person agreement rules:

    The following decision tree visualizes the rules used for classifying presence/absence of morphological agreement between two tokens that are connected by a dependency relation denoted by relation. head-pos and child-pos refer to the POS tag of the head and child token respectively.

    Each node of the tree represents a portion of the data. samples denotes the number of training data points in that node. value is the class distribution within that node. Each edge denotes the feature used for splitting.
    Leaf nodes contain the description of all of the features that appear in that leaf. * denotes that the feature can take any value.

    Given that all feature values for Person are often not equally probable, we use a threshold t to decide the probability of agreement vs non-chance agreement. Click on t=0.9 to toggle between showing/hiding the tree with threshold=0.9

    We evalute the tree on test data along three metrics. Unweighted Distributional Similarity (UDS) measures how the class distribution over the training data matches with the test data for each leaf. Weighted Distributional Similarity (WDS) weighs the UDS score with percent of test data in that leaf. Agreement-Only Distributional Similarity (ADS) measures the UDS score for only the "agreement" class

      Agreement Rules:

    1. All tokens agree with their head tokens for the dependency relations: nominal subject (nsubj)

    2. VERB tokens agree with their head for the dependency relations: conjunct(conj)

    3. PRON tokens agree with their head for the dependency relations: passive nominal subject(nsubj:pass)
